What is the family in the glm command?

  • Generalized linear models have two parts
  • Linear predictor \(\boldsymbol{\theta}^T \boldsymbol{X}_i = \beta_0 + \beta_1 X_{1i} + \ldots + \beta_p X_{pi}\)
  • Link and variance functions \(\mathbb{E}Y_i = \mu_i\) and \(g(\mu_i) = h_i\) and \(Var(Y_i) = \phi V(\mu_i)\)
  • The above doesnât specify the probaility distribution
    • The family gives that distribution

Binomial family

  • Model the binary data with bernoulli distribution (special case of the binomial)

  • \(P(Y_i = y| X_1, X_2, \ldots) = \begin{cases} p_i = p(\boldsymbol{X}_i) & \quad \text{if } y = 1\\ 1 - p_i = 1 - p(\boldsymbol{X}_i) & \quad \text{if } y = 0\\\end{cases}\)

Could you please explain null and residual deviance in details?

  • Deviance is a more generalized goodness of fit metric used in GLMs
    • Consider it as a generalization of the Residual Sum of Squares \(RSS = \sum_{i = 1}^n (Y_i - \widehat{Y_i})^2\)
  • It is based off the log likelihood of the data assuming different model setups.
    • Log likelihood \(\mathcal L(\boldsymbol{\theta}|Y, \boldsymbol{X})\)
  • Null model (Choose \(\boldsymbol{\theta}^T \boldsymbol{X}_i = \beta_0\))
  • Saturated or perfect model has a parameter for each observation
    • \(n\) observations \(\leadsto n\) parameters
    • Overfit the model to give a perfect fit

Ok so what are the null and residual deviance exactly?

  • Goodness of fit measure by looking how much of the likelihood changes
  • Null deviance \(D_0 = -2(\mathcal L (\beta_0) - \mathcal L_s)\)
    • Measures Change in likelihood from perfect fit to probably horrible fit
  • Residual deviance \(D = -2(\mathcal L(\widehat{\boldsymbol{\theta}}) - \mathcal L_s)\)
    • Measures change in likelihood from perfect fit to current model.

Parametric vs Non-parametric

  • Logistic regression has interpretable parameters.

  • \(\log(p/(1-p)) = \beta_0+ \beta_1 X_1 + \ldots + \beta_p X_p\)

  • \(\frac{p}{1 - p} = \frac{P(Y = 1 | \boldsymbol{X})}{P(Y = 0 | \boldsymbol{X})} = \exp \left( \beta_0+ \beta_1 X_1 + \ldots + \beta_p X_p \right)\)

  • Consider change of unit increase in \(X_1\) between two observations

  • \(\frac{\frac{P(Y = 1 | X_1 = x + 1)}{P(Y = 0 | X_1 = x + 1)}}{\frac{P(Y = 1 | X_1 = x)}{P(Y = 0 | X_1 = x)}} = \frac{\exp \left( \beta_0+ \beta_1 (x + 1) + \ldots + \beta_p X_p \right)}{\exp \left( \beta_0+ \beta_1 x + \ldots + \beta_p X_p \right)} = e^{\beta_1}\)

why we can use mean() to calculate the accuracy of classification

    • mean wants numeric input and will convert TRUE to 1 and FALSE to zero.

x == y
## [1]  TRUE  TRUE  TRUE FALSE  TRUE
mean(x == y)
## [1] 0.8
sum(x == y)/length(x)
## [1] 0.8

Bayes, LDA vs QDA

  • Bayes classification
    • Classify according to the rule \(\widehat c = \arg\max_i P(x \in C_i|x) = f(x| x \in C_i) P(x \in C_i)/f(x)\)
    • \(\widehat c = \arg\max_i f_i(x) \pi_i\)
  • LDA assumes normal distributed data with means different for each class
    • A common covariance structure to all classes.
  • QDA assumes normally distributed data (same as above)
    • Each class can have a different covariance matrix
  • Could do naive bayes assume simplifying structure
    • \(f_i(\boldsymbol{x}) = \prod_{j = 1}^d f_{ij}(x_i)\)

Fishers Linear Discriminant

  • Similar to ANOVA
  • Looks at the ratio of between to within variability.
    • \(FC = \frac{(\mu_1 - \mu_2)^2}{s_1^2 + s_2^2}\)
    • \(\mu_i\) are the class means and \(s_i^2\) are the class variances.
  • Looks for a transformation \(w\) that maximizes that ratio
    • \(FC_w = \frac{(\mu_{1w} - \mu_{2w})^2}{s_{1w}^2 + s_{2w}^2}\)
    • Define \(y = \boldsymbol{w}^T \boldsymbol{x} = w_1 x_1 + w_2 x_2 + \ldots w_p x_p\)
